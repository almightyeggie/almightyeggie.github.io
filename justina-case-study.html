Background

The brief: what would it look like?

JustLogin's product team was exploring the integration of an AI assistant — internally named Justina — directly inside their HR SaaS platform. The assistant would allow users to query HR data using natural language, with the system responding in real time.

One specific question the team needed answered before development could move forward: what would it look like when the AI responds with a chart? Could the interface handle rich data responses without breaking the existing UI? And would it scale across screen sizes?

"We need to visualize how charts would look inside the bot response — and it has to be responsive."

My manager brought this to me as a design and prototyping task. There were no existing references, no design system for the AI layer — just a blank canvas within the JustLogin Bootstrap framework.

Process

From brief to working prototype

Rather than producing static mockups, I built a fully interactive HTML prototype that simulated a real conversation flow — including user prompts, bot responses with action cards, and dynamic chart rendering.

1

Scoped the interaction flow

Mapped out a realistic conversation: user selects a module (Leave), asks a question, bot responds with a list and action buttons, user requests a chart, bot renders it — then user asks to change the chart type.

2

Designed within the existing system

Used JustLogin's Bootstrap framework to keep the prototype visually consistent with the live product — same nav, sidebar, color palette, and typography. No new design system needed; the AI layer had to feel native.

3

Integrated Chart.js for dynamic rendering

Built two chart types — a multi-line chart and a grouped bar chart — using Chart.js. Both were fully responsive and switchable within the same conversation thread, demonstrating that the bot could adapt output based on follow-up user requests.

4

Created the Justina bot persona

Adapted a Canva illustration into a branded bot avatar that matched JustLogin's visual identity. The bot name "Justina" was decided collaboratively by the team through a vote.

5

Tested responsiveness

Ensured the entire interface — chat bubbles, action cards, and charts — scaled correctly across desktop, tablet, and mobile breakpoints using Bootstrap's grid system.

Output

The final prototype

A fully coded, responsive HTML prototype demonstrating a complete AI-assisted HR query flow — from natural language input to dynamic chart rendering, with real-time chart type switching within the same conversation.

Justina AI Bot prototype — showing a Leave module query, pending approvals response, and dynamic chart switching from line to bar chart. Built on JustLogin's Bootstrap framework with Chart.js.

The prototype demonstrated two key interactions: the bot surfacing actionable HR data (pending leave approvals with reject/approve buttons), and the bot rendering a full multi-series chart in response to a data query — then switching chart types on user request without leaving the conversation.
